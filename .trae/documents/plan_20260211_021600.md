我将对前端页面进行全面重构，打造一个“AI 全能 Agent”平台。

### 核心设计思路

1. **架构模式**：采用 **首页导航 + 侧边栏切换** 的布局结构。

   * **首页**：展示“编程助手”和“旅游助手”两个精美卡片，作为功能入口。

   * **工作台**：进入具体助手后，左侧显示导航栏方便切换，右侧为沉浸式聊天窗口。
2. **API 适配**：改造前端 API 层，支持根据当前选择的助手类型路由到不同的后端接口（`/api/ai/chat/code`或 `/api/ai/chat/travel`）。
3. **视觉升级**：使用现代化 UI 设计，包括渐变背景、卡片悬浮特效、柔和阴影和更美观的字体。

### 具体实施步骤

#### 1. 升级 API 层 (`src/api/chatApi.js`)

* 修改 `chatWithSSE` 函数，新增 `type` 参数。

* 根据 `type` 动态构建请求 URL：

  * `code` -> `/api/ai/chat/code`

  * `travel` -> `/api/ai/chat/travel`

#### 2. 重构主应用组件 (`src/App.vue`)

* **状态管理**：新增 `currentMode` 状态，取值为 `'home'` (首页), `'code'` (编程助手), `'travel'` (旅游助手)。

* **模板结构改造**：

  * **首页区域** (`v-if="currentMode === 'home'"`):

    * 大标题：“AI 全能 Agent”。

    * 双卡片布局：

      * 🔵 **编程助手**：图标 💻，描述“解答技术难题，提供代码示例”。

      * 🟢 **旅游助手**：图标 ✈️，描述“规划完美行程，探索世界之美”。

  * **工作台区域** (`v-else`):

    * **侧边栏**：显示当前助手列表，支持点击切换，以及“返回首页”按钮。

    * **聊天区**：复用现有的聊天组件，但根据 `currentMode` 动态展示不同的：

      * 欢迎语（例如：“我是你的旅游规划师...” vs “我是你的编程助手...”）。

      * 头像（🤖 vs ✈️）。(更改机器人头像，下载地址<https://preview.qiantucdn.com/freepik/512/4681/4681147.png!qt_h320_webp>)

      * API 调用逻辑。

* **样式美化**：

  * 引入 CSS 变量定义主题色。

  * 增加 Flex/Grid 布局，确保响应式适配。

  * 优化滚动条和按钮样式。

#### 3. 验证与测试

* 验证首页卡片点击是否正确跳转。

* 验证侧边栏切换功能。

* 验证两种模式下是否分别调用了正确的后端接口（通过观察回复内容和网络请求）。

此方案将保留所有现有功能（Markdown 渲染、流式输出、错误处理），同时显著提升用户体验和界面美观度。
